# 三角洲游戏助手 - 小麻虾电竞出品

开源、绿色、无毒、免登录，Windows端专属游戏辅助工具。

## 📋 功能特性

### 核心功能
- 🎯 **屏幕中心准星** - 强制置顶，穿透输入，自定义颜色、大小、发光效果
- 📚 **AI Skill技能库** - 内置战备配置和撤离点攻坚攻略
- 🔒 **一键断上传** - 仅断开上传，不影响游戏下载、语音等下行功能
- 🔄 **提示位置切换** - 左上角、右上角、屏幕底部三个位置可选

### 选配功能
- 🤖 **RGA AI问答** - 本地向量检索+智能问答（需手动配置启用）

## 🚀 快速开始

### 环境要求
- Windows 7/8/10/11
- Python 3.8+

### 依赖安装

#### 基础依赖（必装）
```bash
pip install PySide6
```

#### 选配AI问答模块依赖（可选）
```bash
pip install chromadb sentence-transformers
```

### 运行程序
```bash
python main.py
```

## 📦 PyInstaller打包

### 基础版打包（不含AI问答模块）
```bash
pyinstaller --onefile --windowed --name "三角洲游戏助手" --icon=icon.ico main.py
```

### 完整版打包（含AI问答模块）
```bash
pyinstaller --onefile --windowed --name "三角洲游戏助手_完整版" --icon=icon.ico --add-data "skills.json;." main.py
```

**打包参数说明：**
- `--onefile`: 打包成单个EXE文件
- `--windowed`: 不显示控制台窗口
- `--name`: 指定输出文件名
- `--icon`: 指定图标文件（可选）
- `--add-data`: 包含数据文件

## 📖 使用说明

### 1. 准星设置
- **快捷键**: F9（可在设置中修改）
- 支持自定义：
  - 准星颜色
  - 准星大小（10-50像素）
  - 外发光效果（颜色、半径）
- 窗口属性：强制置顶、透明背景、穿透鼠标输入

### 2. AI Skill技能库
内置两大分类：
- **⚔️ 战备配置**
  - 跑刀流配装
  - 以小搏大流派（修脚枪专属）
- **💥 撤离点攻坚**
  - 露娜炸撤离点点位
  - 红狼炸撤离点点位
  - 乌鲁鲁炸撤离点点位

### 3. 一键断上传
- **快捷键**: F10（可在设置中修改）
- 仅阻断网络上传，不影响下载、语音等下行功能
- 操作后显示3秒强制置顶提示
- 提示位置可切换：左上角/右上角/屏幕底部

### 4. AI问答模块（选配）

#### 启用步骤
1. 打开「设置」Tab页
2. 勾选「启用AI问答模块」
3. 重启工具
4. 首次启用自动检测并安装依赖
5. 自动初始化Chroma向量数据库

#### 使用说明
- 启用后「AI问答」Tab页正常显示
- 输入问题自动检索本地知识库
- 仅在提问时触发检索，无后台资源消耗
- 可随时在设置中关闭，释放资源

## 🔧 配置说明

## 📝 更新日志

### 2026-03-17
- **全局热键实现**：将热键处理从 PySide6 的 `QShortcut` 改为使用 Windows API 的全局热键
- **依赖项更新**：添加了 `ctypes` 库依赖（Python标准库，无需额外安装）
- **热键响应优化**：确保窗口失去焦点时热键仍能正常响应
- **热键注销机制**：程序关闭时自动注销全局热键，避免热键冲突
- **准星显示优化**：
  - 修复准星位置计算，使用完整屏幕尺寸定位到屏幕中心
  - 增强准星置顶机制，确保在游戏窗口打开时仍能显示在最前面
  - 添加定期位置更新和置顶检查，防止准星位置偏移

**更改原因**：
- **窗口失去焦点时热键失效问题**：之前使用 `QShortcut` 注册的热键只在窗口有焦点时有效，当用户切换到游戏或其他应用时，F9和F10热键无法响应
- **全局热键需求**：游戏辅助工具需要在游戏运行时也能响应热键，必须使用全局热键
- **准星位置偏移问题**：打开游戏时准星位置会变动到游戏右下部分，需要确保准星始终定位到屏幕中心
- **准星显示问题**：游戏窗口可能会覆盖准星窗口，需要增强置顶机制
- **涉及代码**：
  - `ui/main_window.py` 中的 `init_hotkeys` 方法：使用 Windows API 的 `RegisterHotKey` 函数注册全局热键
  - `ui/main_window.py` 中的 `nativeEvent` 方法：监听 Windows 消息，捕获全局热键事件
  - `ui/main_window.py` 中的 `closeEvent` 方法：程序关闭时使用 `UnregisterHotKey` 注销热键
  - `widgets/crosshair.py` 中的 `update_position` 方法：使用 `screen.geometry()` 获取完整屏幕尺寸
  - `widgets/crosshair.py` 中的 `setup_force_topmost` 方法：添加定期位置更新和置顶检查
- **解决方案**：
  - 使用 Windows API 的 `RegisterHotKey` 函数注册全局热键，确保热键在系统级别有效
  - 实现 `nativeEvent` 方法，监听 Windows 的 `WM_HOTKEY` 消息
  - 在程序关闭时注销全局热键，避免热键冲突和资源泄漏
  - F9（虚拟键码0x78）用于切换准星，F10（虚拟键码0x79）用于切换断上传
  - 修改准星位置计算，使用完整屏幕尺寸定位到屏幕中心
  - 添加定期位置更新和置顶检查，确保准星始终显示在屏幕中心且置顶

### 2026-03-16
- **热键处理优化**：将热键处理从使用 `keyboard` 库改为使用 PySide6 的 `QShortcut`
- **依赖项更新**：移除了 `keyboard` 库依赖，只需要安装 `PySide6`
- **入口文件更新**：将入口文件从 `deforce_helper.py` 改为 `main.py`
- **运行和打包命令更新**：相应更新了运行和打包命令
- **防火墙规则处理优化**：添加了规则检查、错误处理和管理员权限提示
- **防重复触发机制**：添加了冷却时间，防止快速连续触发热键导致程序卡死

**更改原因**：
- **热键不响应问题**：之前使用 `keyboard` 库在子线程中处理热键事件，导致线程安全问题，子线程中执行 UI 操作会导致 PySide6 应用程序崩溃或无响应
- **程序卡死问题**：快速连续触发热键时，会导致多个线程同时执行网络操作，网络操作（断上传/恢复上传）是阻塞操作，会导致 UI 线程被阻塞
- **涉及代码**：
  - `ui/main_window.py` 中的 `init_hotkeys` 方法：使用 `keyboard` 库在子线程中注册热键
  - `ui/main_window.py` 中的 `toggle_crosshair_hotkey` 和 `toggle_upload_block` 方法：直接在子线程中执行 UI 操作
  - `ui/main_window.py` 中的 `_toggle_crosshair_ui` 和 `_toggle_upload_block_ui` 方法：执行阻塞的网络操作
- **解决方案**：
  - 使用 PySide6 的 `QShortcut` 代替 `keyboard` 库，确保在主线程中处理热键事件
  - 优化执行逻辑，添加防重复触发机制
  - 使用 `QTimer.singleShot` 处理冷却时间，避免快速连续触发
- 简化依赖项，减少安装步骤
- 统一入口文件命名，提高代码可维护性
